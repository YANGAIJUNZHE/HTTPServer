06. 并发请求

在服务器端网络编程中处理并发连接有三种方式：

分别是

1）分叉（forking）: fork为每个客户连接创建新的进程以实现并发

2）多线程: 则使用线程代替进程

3）事件循环。使用轮询和非阻塞 IO。

由于fork进程消耗太大，已经不提倡。

多线程比进程轻量，曾广泛应用于服务器，但是由于NIO（非阻塞IO）的进程和线程的开销表现，大多数现代生产级软件使用事件循环进行网络编程。


以下示例仍然使用线程来阐述，同学可以根据自己需要改成NIO的模型

一、服务器端

在处理部分使用了pthread_create来创建线程，创建后继续循环处理下个请求。

1）main函数：

while (true) {
        // accept
        struct sockaddr_in client_addr = {};
        socklen_t addrlen = sizeof(client_addr);
        int connfd = accept(fd, (struct sockaddr *) &client_addr, &addrlen);
        if (connfd < 0) {
            perror("创建连接失败!");
            continue; // error
        }

        pthread_t pid;
        pthread_create(&pid, NULL, process_long_request, (void *) connfd);
        pthread_detach(pid);
    }

2）process_long_request方法

void *process_long_request(void *arg) {
    int conn_fd = (int) arg;

    while (true) {
        int32_t err = process_on_request(conn_fd);
        if (err) {
            break;
        }
    }

    close(conn_fd);
}
